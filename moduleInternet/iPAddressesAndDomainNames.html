IP-адреси та доменні імена:

  1. Що таке IP-адреса і для чого вона використовується?
  2. Яка різниця між IPv4 та IPv6 адресами?
  3. Яка максимальна кількість IP-адрес, яку може надати IPv4? Яка максимальна кількість IP-адрес, яку може надати IPv6?
  4. Яка різниця між публічними та приватними IP-адресами?
  5. Які діапазони IP-адрес використовуються для локальних мереж?
  6. Чому адреса 127.0.0.1 відома як "адреса localhost"?

1. Що таке IP-адреса і для чого вона використовується?

IP-адреса – це унікальний ідентифікатор, який призначається кожному пристрою в мережі для забезпечення комунікації. Вона використовується для ідентифікації та локалізації пристроїв у мережі.

2. Яка різниця між IPv4 та IPv6 адресами?

IPv4 використовує 32-бітні адреси, тоді як IPv6 використовує 128-бітні адреси. Це призводить до значного збільшення кількості можливих адрес у IPv6 порівняно з IPv4.

3. Яка максимальна кількість IP-адрес, яку може надати IPv4? Яка максимальна кількість IP-адрес, яку може надати IPv6?

IPv4 може надати приблизно 4,294,967,296 (2^32) унікальних IP-адрес. IPv6, з іншого боку, може надати приблизно 3.4 x 10^38 (2^128) унікальних IP-адрес, що є величезною кількістю порівняно з IPv4.

4. Яка різниця між публічними та приватними IP-адресами?

Публічні IP-адреси призначені для ідентифікації пристроїв у глобальній мережі Інтернет, тоді як приватні IP-адреси використовуються для ідентифікації пристроїв у межах приватної мережі (наприклад, у домашніх або корпоративних мережах).

5. Які діапазони IP-адрес використовуються для локальних мереж?

Діапазони IP-адрес, які використовуються для локальних мереж, зазвичай визначаються такими блоками:

  • IPv4: 10.0.0.0 до 10.255.255.255, 172.16.0.0 до 172.31.255.255, 192.168.0.0 до 192.168.255.255
  • IPv6: fc00::/7 (Unique Local Addresses)

6. Чому адреса 127.0.0.1 відома як "адреса localhost"?

Адреса 127.0.0.1 призначена для тестування мережевого з'єднання пристрою з самим собою. Ім'я "localhost" асоціюється з цією адресою для зручності. Коли комп'ютер спробує встановити з'єднання з localhost, він звертається до власної мережевої карти, що дозволяє тестувати мережеві додатки локально.
